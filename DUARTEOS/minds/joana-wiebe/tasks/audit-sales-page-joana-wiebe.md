# Task: Auditar Sales Page (Joanna Wiebe Method)

## Objetivo
Conduzir auditoria completa de uma sales page existente usando a metodologia Joanna Wiebe — Green/Red/Yellow system + Seven Sweeps + framework analysis. "Refuse to rewrite immediately."

## Processo

### Fase 1: Performance Data
**Antes de qualquer opiniao, colete dados:**
1. Conversion rate atual
2. Bounce rate
3. Scroll depth (onde pessoas param)
4. Click maps / heat maps
5. A/B test results anteriores (se existem)
6. Revenue atribuida a pagina

### Fase 2: Green/Red/Yellow Audit

**Green (manter):**
- Elementos com bom engagement (scroll, clicks)
- Sections com social proof efetivo
- CTAs que convertem
- Headlines que reteem

**Red (consertar):**
- Headline generica / vaga
- Copy company-centric ("We are..." "Our product...")
- Ausencia de social proof
- CTAs genéricos ("Submit", "Click here")
- Design liderando copy (lorem ipsum)
- Stage of awareness wrong no hero
- FAQs escondidos em accordions com informacao critica

**Yellow (otimizar):**
- Especificidade insuficiente (beneficios abstratos)
- Message hierarchy subotima
- CTV vs CTA mal posicionados
- Agitation insuficiente
- Proof inadequado (claims sem evidencia)
- Zero-risk ausente (sem garantias ou reversals)

### Fase 3: Framework Analysis
Verifique se a pagina segue (ou deveria seguir):
- [ ] 10/90 Rule — top 10% matcha awareness stage?
- [ ] Seven-Question hierarchy — messaging em ordem logica?
- [ ] Rule of One — foco em um leitor, oferta, idea, promessa?
- [ ] PCPO — Problem, Cure, Proof, Offer presentes?
- [ ] PAS — agitation suficiente antes da solucao?

### Fase 4: Recomendacoes Priorizadas
Organize por impacto:
1. **Quick wins** — mudancas de copy que podem ser testadas imediatamente
2. **Strategic changes** — restructuracao de messaging hierarchy
3. **Full rewrite candidates** — secoes que precisam ser refeitas do zero
4. **Research gaps** — onde falta VOC data para tomar decisao

### Fase 5: Apresentacao
Apresente findings ao stakeholder com:
- Mapa visual Green/Red/Yellow
- Prioridades de mudanca com impacto estimado
- Recomendacao: testar A/B o quick win de maior impacto primeiro

## Output Esperado
- Documento de auditoria com Green/Red/Yellow annotations
- Lista priorizada de mudancas (quick wins → strategic → rewrites)
- Gaps de pesquisa identificados
- Recomendacao de primeiro teste A/B
- Wireframe de messaging hierarchy corrigida

## Checklist de Qualidade
- [ ] Performance data coletada ANTES de opinar?
- [ ] Green/Red/Yellow aplicado a cada secao?
- [ ] Framework analysis completa (10/90, Seven-Q, Rule of One, PCPO)?
- [ ] Recomendacoes priorizadas por impacto?
- [ ] Findings apresentados antes de qualquer mudanca?
- [ ] Primeiro teste A/B recomendado?
- [ ] Preferencia pessoal NAO influenciou auditoria?
