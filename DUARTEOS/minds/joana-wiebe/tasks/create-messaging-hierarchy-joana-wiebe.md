# Task: Criar Messaging Hierarchy (Joanna Wiebe Method)

## Objetivo
Construir a hierarquia de messaging completa para um produto/servico usando Message Map, Seven-Question framework, Three Sales Arguments, e stages of awareness. Este e o "messy middle" — o trabalho de pensar que o cliente realmente paga.

## Processo

### Fase 1: Research (The Messy Middle)
1. **VOC Data Collection** — review mining, surveys, interviews, demo replays
2. **The Ultimate Question** — "What was going on in your life that brought you to [produto]?"
3. **Notas cruas** — Apple Notepad, minimal formatting, aspas para citacoes diretas, ALL CAPS para enfase
4. **Permitir que as notas sejam intencionalmente messy** — o pensamento acontece por imersao

### Fase 2: Message Map Construction
Organize research em:

**Product Section:**
- O que e, quem fez, como funciona
- Features e especificacoes
- Cost e value proposition
- Minimum viable commitments

**Persona Sections (por audiencia):**
- Problemas principais (ranked por frequencia + intensidade)
- Motivadores de acao
- Momentos de maior tensao
- Solucoes ja tentadas (e por que falharam)
- Outcomes desejados (linguagem deles, nao sua)

**Jobs to Be Done:**
- Switching habits
- Worries (preocupacoes)
- Pushes (forcas que empurram para fora do status quo)
- Pulls (forcas que puxam para sua solucao)

**Conversion Elements:**
- Crencas necessarias para dizer sim
- Precursores de conversao
- Criterios de compra
- Concorrencia percebida
- Restricoes de decisao
- Click triggers

### Fase 3: Three Sales Arguments
1. **Current state** — O que sabem, sentem, ja acreditam?
2. **Required beliefs** — O que precisam acreditar para dizer sim?
3. **Beliefs to reframe** — Que crencas existentes impedem conversao?

### Fase 4: Hierarchy Design
Usando Seven-Question framework, defina a sequencia:
1. O que fazemos (match awareness stage)
2. Por que voce deveria se importar (conexao problema-solucao)
3. Outros se importam (social proof)
4. Como funciona (processo/demonstracao)
5. Como sua vida melhora (outcome painting)
6. Por que e seguro acreditar (risk reversal)
7. Proximo passo (CTA)

### Fase 5: Rule of One Validation
- [ ] UM leitor definido?
- [ ] UMA oferta clara?
- [ ] UMA big idea singular?
- [ ] UMA promessa especifica?

## Output Esperado
- Message Map completo (Product + Persona + JTBD + Conversion)
- Three Sales Arguments respondidas
- Hierarchy de messaging em ordem (Seven-Question)
- Rule of One definido
- Stage of awareness primario identificado
- Top 5 headlines candidates (baseados em VOC)

## Checklist de Qualidade
- [ ] VOC data suficiente coletada?
- [ ] Message Map completo (todas as secoes)?
- [ ] Three Sales Arguments respondidas?
- [ ] Seven-Question hierarchy definida?
- [ ] Rule of One validado?
- [ ] Linguagem do cliente preservada (nao reescrita)?
- [ ] Stage of awareness identificado?
- [ ] Crencas limitantes mapeadas para reframing?
